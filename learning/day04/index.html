Semantic HTML is the use of HTML markup to reinforce the meaning of the information in the web page, rather than just to define its presentation or look.

Examples of semantic elements include:

  • <header> - Defines the header of a page or section
  • <nav> - Defines the navigation links of a page
  • <main> - Defines the main content of a page
  • <section> - Defines a section in a page
  • <article> - Defines an independent piece of content
  • <footer> - Defines the footer of a page or section


Semantic HTML provides several benefits:

  • Better Readability: The structure of the page is more meaningful and easier to understand.
  • Better SEO: Search engines can better understand the content and structure of the page.
  • Better Accessibility: Screen readers and other assistive technologies can better interpret the content.
  • Easier Maintenance: The code is more maintainable and easier to update.


Here are some common semantic HTML elements and their purposes:

  • Header: The header tag is used to define the top section of a webpage, which typically contains the website's logo, title, and navigation links. It provides a consistent look and feel across different pages of the site.
  • Nav: The nav tag is used to define a section of navigation links. It helps users easily navigate through different sections or pages of the website, improving user experience.
  • Main: The main tag is used to encapsulate the primary content of a webpage. It ensures that screen readers and search engines can identify the main content area, enhancing accessibility and SEO.
  • Section: The section tag is used to group related content together within a webpage. It can be used for different topics or themes, making it easier for users to understand the structure of the content.
  • Article: The article tag is used to define independent pieces of content that can stand alone, such as blog posts, news articles, or forum posts. Each article can have its own heading and content.
  • Footer: The footer tag is used to define the bottom section of a webpage, which typically contains copyright information, contact details, and links to privacy policies or terms of service. It provides closure to the page's content.
